# cortex-skill

Connect your local Cortex knowledge infrastructure to any AI agent.

## What it does

- **Ingest**: Send links, articles, and text notes into Cortex for processing
- **Inbox**: View and act on Cortex notifications (read / ack / dismiss)
- **Feedback**: Rate signals (useful / not_useful / wrong / save_for_later)
- **Push**: Receive Cortex webhook notifications via a local relay

## Installation

### One-command install

```bash
git clone https://github.com/MatteoCui001/cortex-skill.git ~/Projects/cortex-skill && python3 ~/Projects/cortex-skill/scripts/bootstrap_local_cortex.py
```

This single command:
1. Clones the skill repo to `~/Projects/cortex-skill`
2. Clones or updates the Cortex engine at `~/Projects/cortex`
3. Runs `install.sh` — handles all dependencies automatically:
   - Homebrew, PostgreSQL 16, pgvector (source build), uv, Python 3.12
   - Database creation, 9 migrations (pgvector/zhparser optional)
   - Embedding model download, web console build
   - API token generation in `~/.cortex/env`
4. Writes `~/.cortex/skill_config.yaml` with connection config + API token
5. Auto-registers as a skill in detected agent platforms:
   - `~/.claude/skills/cortex` (Claude Code)
   - `~/.openclaw/skills/cortex` (OpenClaw)
   - `~/.codex/skills/cortex` (Codex)
   - `~/.agents/skills/cortex` (generic)
6. Creates a launchd plist for auto-start on login
7. Starts Cortex and waits for health check

Takes ~5-10 minutes on first run. Idempotent (safe to re-run).
After completion, Cortex is running at `http://127.0.0.1:8420`.

### Verify installation

```bash
python3 ~/Projects/cortex-skill/scripts/doctor.py
```

## Commands

The command router accepts plain text input and returns JSON. Any agent can call it:

```bash
# Via argv
python3 scripts/command_router.py "some text to ingest"

# Via stdin
echo '{"text": "inbox"}' | python3 scripts/command_router.py
```

| Input | Action |
|-------|--------|
| Any URL | `POST /events/ingest` with URL |
| Any text | `POST /events/ingest` as note |
| `inbox` or `收件箱` | `GET /notifications` |
| `read <id>` | `POST /notifications/{id}/read` |
| `ack <id>` | `POST /notifications/{id}/ack` |
| `dismiss <id>` | `POST /notifications/{id}/dismiss` |
| `useful <id>` / `not_useful <id>` | `POST /signals/{id}/feedback` |

## Architecture

```
Any Agent (Claude Code / OpenClaw / Codex / ...)
         |
         v
    Skill (command_router.py)
         |
         v
    Local Cortex API (127.0.0.1:8420)
         |
         v (webhook, optional)
    Local Relay (127.0.0.1:8421)
         |
         v
    Agent push notification
```

## Service management

Cortex runs as a launchd service (auto-starts on login):

```bash
# Check status
launchctl list | grep cortex

# Stop
launchctl unload ~/Library/LaunchAgents/com.cortex.serve.plist

# Start
launchctl load ~/Library/LaunchAgents/com.cortex.serve.plist

# Logs
tail -f ~/.cortex/cortex.log
```

## Configuration

After bootstrap, config is stored at `~/.cortex/skill_config.yaml`:

```yaml
cortex:
  base_url: http://127.0.0.1:8420/api/v1
  api_token: "<auto-generated>"
  workspace: default
relay:
  port: 8421
  enabled: false
```
